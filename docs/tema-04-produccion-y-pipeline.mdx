# Tema 4 - Produccion y pipeline

## Objetivos

- Diseñar un pipeline de trabajo reproducible.
- Integrar control de versiones y flujo de ramas.
- Asegurar calidad tecnica en cada entrega.

## Contenidos

- Estructura de repositorio y convenciones.
- Flujo de trabajo con ramas y revisiones.
- Integracion continua basica para builds.
- Gestion de incidencias y deuda tecnica.

## Actividades

1. Definir norma de ramas y mensajes de commit.
2. Crear pipeline basico de validacion.
3. Configurar checklist de revision de entregables.
4. Informe de mejoras tecnicas para la siguiente iteracion.

## Evaluacion

- Robustez del pipeline propuesto.
- Cumplimiento de convenciones del equipo.
- Calidad de las evidencias tecnicas.

## Recursos

- Guia de buenas practicas de repositorio.
- Ejemplos de workflows CI para proyectos de juegos.
- Plantilla de checklist de calidad.
