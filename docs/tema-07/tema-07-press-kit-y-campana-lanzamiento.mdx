# Tema 7.2. Press kit y campaña de lanzamiento

El ***press kit*** (*kit de prensa*) es el paquete de información y assets que envías a medios,
*influencers* y curadores de plataformas para que hablen de tu juego. La **campaña de
lanzamiento** coordina cuándo y dónde publicas cada pieza de comunicación.

## Objetivos

- Crear un *press kit* mínimo viable profesional.
- Redactar una nota de prensa clara y orientada al público objetivo.
- Diseñar un calendario de comunicación de lanzamiento realista.
- Adaptar el mensaje a redes sociales y comunidad.

## Contenidos

### 1. Estructura del press kit

Un *press kit* indie estándar incluye:

| Elemento | Contenido |
| --- | --- |
| **Fact sheet** | Nombre, género, plataforma, fecha, precio, clasificación PEGI, contacto |
| **Descripción corta** | 1–2 frases (*elevator pitch*) |
| **Descripción larga** | 2–3 párrafos: historia, mecánicas, público |
| **Características clave** | Lista de *x-features* (del [Tema 2](../tema-02/tema-02-game-design-fundacional.mdx)) |
| **Capturas** | HD sin UI excesiva; variedad de situaciones |
| **Trailer / GIFs** | Enlace a vídeo de 30–90 s |
| **Logos y assets** | Logo en PNG transparente, *key art* si existe |
| **Créditos** | Equipo, licencias de música, motor |
| **Enlaces** | Web, Steam/itch, redes, trailer |

Formato de entrega: carpeta ZIP, página web (`presskit.html`) o herramientas como
[presskit()](https://dopresskit.com/) (referencia de sector).

### 2. Nota de prensa

La **nota de prensa** es un texto para medios con estructura clásica:

1. **Titular**: nombre del juego + gancho ("lanza demo", "fecha confirmada").
2. **Lead**: quién, qué, cuándo, dónde en el primer párrafo.
3. **Cuerpo**: qué lo hace especial, citas del equipo (opcional).
4. **Boilerplate**: una línea sobre el estudio o el proyecto académico.
5. **Contacto**: email para prensa.

Longitud orientativa: 300–500 palabras.

### 3. Plan editorial y comunidad

Calendario típico **4–6 semanas** antes del lanzamiento (escala según recursos):

| Semana | Acción |
| --- | --- |
| T−6 | Anunciar fecha; publicar *key art* o captura fuerte |
| T−4 | Trailer o gameplay de 60 s; abrir lista de deseos (Steam) |
| T−2 | Devlog o *behind the scenes*; contacto con prensa indie |
| T−1 | Recordatorio de fecha; stream o preview a *creators* |
| T0 | Lanzamiento; nota de prensa; post en todas las redes |
| T+1 | Agradecimiento; recopilar valoraciones y *feedback* |

Canales habituales para indie:

- **Twitter/X**, **Bluesky**, **Mastodon** (comunidad dev).
- **Discord** del proyecto o del género.
- **Reddit** (subreddits del género, con normas anti-spam).
- **TikTok / YouTube Shorts** (clips cortos de gameplay).

:::tip[Un mensaje, muchos formatos]
La frase de posicionamiento del [Tema 1](../tema-01/tema-01-aplicacion-proyecto-2d.mdx) debe
aparecer en fact sheet, nota de prensa y biografía de tienda. **Coherencia** > volumen.
:::

### 4. Press kit mínimo viable (curso)

Para la entrega del módulo, el mínimo aceptable:

- Fact sheet completo.
- Descripción corta y larga.
- 4+ capturas en buena resolución.
- Enlace a trailer (del [Tema 8](../tema-08/tema-08-captura-edicion-e-trailer.mdx) o fase anterior).
- Logo o imagen representativa.
- Contacto del equipo.

## Actividades

1. ***Press kit* mínimo viable**
   - Monta el kit con la estructura de este documento.
   - *Evidencia*: ZIP o enlace público + checklist de elementos.
2. **Calendario de comunicación de lanzamiento**
   - Plan de 4 semanas con fechas, canal y contenido por publicación.
   - *Evidencia*: tabla o calendario en el GDD o documento de marketing.

## Evaluación

| Criterio | Insuficiente | Adecuado | Excelente |
| --- | --- | --- | --- |
| Completitud del press kit | Faltan piezas clave | Mínimo viable completo | Kit profesional y pulido |
| Claridad del mensaje | Confuso o genérico | Mensaje claro | Alineado con posicionamiento |
| Calendario | Irreal o vacío | 4 semanas con acciones | Calendario ejecutable y coherente |
| Adecuación al público | No dirigido | Público considerado | Canales y tono correctos |
